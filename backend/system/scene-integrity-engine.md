## Scene Integrity Engine

**Purpose**: Eliminate static scenes, travelogues, and consequence-free chapters by enforcing a universal scene structure and outcome.

---

## Core Requirements

Every scene must include:
1. **Goal**: What the POV character wants in this scene.
2. **Obstacle**: The resistance that blocks or complicates the goal.
3. **Choice**: The decision made in response to the obstacle.
4. **Result**: A clear outcome that changes the story state.

Every scene must end with **one explicit consequence**:
- **Cost**: Something lost (resource, status, opportunity, relationship).
- **Gain**: Something obtained (information, ally, advantage, tool).
- **Obligation**: A new commitment, risk, or constraint introduced.

---

## Scene Function Tagging

Each scene must be tagged with one primary function:
- **Setup**
- **Escalation**
- **Reveal**
- **Reversal**
- **Payoff**

Rules:
- No more than **one** scene with the same function in a row.
- A repeated function requires **escalation** (greater stakes or consequences).

---

## Failure Triggers (Hard Stops)

- Missing any of Goal / Obstacle / Choice / Result.
- No explicit consequence at scene end.
- Scene does not change the story state.
- Duplicate scene function without escalation.

---

## Verification Checklist

- [ ] Goal is stated or implied and specific.
- [ ] Obstacle is concrete and non-trivial.
- [ ] Choice is a decision, not a reaction.
- [ ] Result alters future options.
- [ ] Consequence is explicit (cost, gain, or obligation).
- [ ] Scene function tag is assigned and not duplicated without escalation.

---

## Notes on Universal Applicability

This engine applies to all genres and all narrative modes. Quiet scenes are allowed, but they must still contain consequence and story-state change.
